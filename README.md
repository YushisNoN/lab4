## Язык программирования
### Синтаксис
Язык представляет собой **ассемблер** с поддержкой:
- Меток (labels)
- Секций (`.text`, `.data`)
- Директивы `.org` для размещения кода по конкретным адресам
- Констант и строк (`.str`)

#### Формат инструкций
| Формат | Синтаксис | Пример |
|--------|-----------|--------|
| R-тип | `OPCODE Rd, Rs1, Rs2` | `ADD R1, R2, R3` |
| I-тип | `OPCODE Rd, Rs1, #imm` | `ADDI R1, R2, #10` |
| J-тип | `OPCODE label` | `JMP loop` |
| O-тип | `OPCODE` | `HALT` |

#### Поддерживаемые инструкции

| Мнемоника | OPCODE | Тип | Операнды | Схема выполнения |
|-----------|--------|-----|----------|------------------|
| `NOP` | 0x00 | O | - | `PC <- PC + 1` — ничего не делает, только инкрементирует счётчик команд |
| `HALT` | 0x01 | O | - | `halt_flag <- 1` — устанавливает флаг остановки, процессор прекращает выполнение |
| `NEG` | 0x02 | R | Rd, Rs1 | `ALU <- -Rs1`; `Rd <- ALU`; `flags <- ALU` — инвертирует знак числа в Rs1, результат в Rd |
| `LOAD` | 0x10 | R | Rd, Rs1, [Rs2] | `addr <- Rs1 + Rs2`; `Rd <- MEM[addr]` — загрузка из памяти по адресу, вычисленному как сумма двух регистров |
| `LOADI` | 0x11 | I | Rd, #imm | `Rd <- imm` — запись непосредственного значения (константы) в регистр |
| `STORE` | 0x12 | R | Rs1, [Rs2] | `addr <- Rs1 + Rs2`; `MEM[addr] <- Rs1` — сохранение значения регистра в память по адресу Rs1+Rs2 |
| `ADD` | 0x20 | R | Rd, Rs1, Rs2 | `ALU <- Rs1 + Rs2`; `Rd <- ALU`; `flags <- ALU` — сложение двух регистров |
| `SUB` | 0x21 | R | Rd, Rs1, Rs2 | `ALU <- Rs1 - Rs2`; `Rd <- ALU`; `flags <- ALU` — вычитание регистров |
| `MUL` | 0x22 | R | Rd, Rs1, Rs2 | `ALU <- Rs1 * Rs2`; `Rd <- ALU`; `flags <- ALU` — умножение (младшие 32 бита) |
| `DIV` | 0x23 | R | Rd, Rs1, Rs2 | `ALU <- Rs1 / Rs2`; `Rd <- ALU`; `flags <- ALU` — целочисленное деление |
| `ADDI` | 0x24 | I | Rd, Rs1, #imm | `ALU <- Rs1 + imm`; `Rd <- ALU`; `flags <- ALU` — сложение с константой |
| `SUBI` | 0x25 | I | Rd, Rs1, #imm | `ALU <- Rs1 - imm`; `Rd <- ALU`; `flags <- ALU` — вычитание константы |
| `AND` | 0x40 | R | Rd, Rs1, Rs2 | `ALU <- Rs1 & Rs2`; `Rd <- ALU`; `flags <- ALU` — побитовое И |
| `OR` | 0x41 | R | Rd, Rs1, Rs2 | `ALU <- Rs1 \| Rs2`; `Rd <- ALU`; `flags <- ALU` — побитовое ИЛИ |
| `XOR` | 0x42 | R | Rd, Rs1, Rs2 | `ALU <- Rs1 ^ Rs2`; `Rd <- ALU`; `flags <- ALU` — побитовое XOR |
| `MOV` | 0x30 | R | Rd, Rs1 | `Rd <- Rs1` — копирование значения одного регистра в другой (без ALU) |
| `CMP` | 0x31 | R | Rs1, Rs2 | `ALU <- Rs1 - Rs2`; `flags <- ALU` — сравнение, результат не сохраняется, только флаги |
| `JMP` | 0x50 | J | label | `PC <- imm` — безусловный переход на адрес из поля imm |
| `JZ` | 0x51 | J | label | `if (Z == 1) PC <- imm` — переход, если флаг Z=1 |
| `JNZ` | 0x52 | J | label | `if (Z == 0) PC <- imm` — переход, если флаг Z=0 |
| `JG` | 0x53 | J | label | `if (Z == 0 && N == V) PC <- imm` — переход, если больше (знаковое сравнение) |
| `JL` | 0x54 | J | label | `if (N != V) PC <- imm` — переход, если меньше (знаковое сравнение) |
| `JGE` | 0x55 | J | label | `if (N == V) PC <- imm` — переход, если больше или равно (знаковое сравнение) |
| `PUSH` | 0x70 | O | - | `SP <- SP - 1`; `MEM[SP] <- Rs1` — стек растёт вниз, запись регистра в стек |
| `POP` | 0x71 | O | - | `Rd <- MEM[SP]`; `SP <- SP + 1` — чтение из стека, стек растёт вверх |
| `CALL` | 0x72 | J | label | `SP <- SP - 1`; `MEM[SP] <- PC`; `PC <- imm` — сохранение адреса возврата, переход на подпрограмму |
| `RET` | 0x73 | O | - | `PC <- MEM[SP]`; `SP <- SP + 1` — восстановление адреса возврата из стека |
| `PSTR` | 0x74 | R | Rs1 | `length <- MEM[Rs1]`; `for i in 0..length-1: OUT <- MEM[Rs1 + i + 1]` — вывод Pascal-строки |
| `IN` | 0x75 | R | Rd | `Rd <- MEM[0xFF]` — чтение из порта ввода (MMIO) |
| `OUT` | 0x76 | R | Rs1 | `MEM[0xFE] <- Rs1` — запись в порт вывода (MMIO) |

#### Пояснение к выполнению инструкций

Все инструкции выполняются за одну или несколько микроопераций. Процессор имеет следующие основные компоненты:

| Компонент | Обозначение | Назначение |
|-----------|-------------|------------|
| Регистровый файл | `RegFile` | 16 регистров R0-R15, R0 всегда равен 0 |
| Указатель стека | `SP` | Хранит адрес вершины стека (растёт вниз) |
| Счётчик команд | `PC` | Хранит адрес текущей инструкции |
| Регистр инструкций | `IR` | Хранит текущую выполняемую инструкцию |
| АЛУ | `ALU` | Выполняет арифметические и логические операции |
| Флаги | `Z, N, V, C` | Устанавливаются по результату операций в ALU |

**Флаги процессора:**

| Флаг | Название | Условие установки |
|------|----------|-------------------|
| `Z` | Zero | Результат операции равен 0 |
| `N` | Negative | Старший бит результата = 1 |
| `V` | Overflow | Переполнение при знаковой операции |
| `C` | Carry | Перенос при беззнаковой операции |

## Организация памяти

Архитектура — **Гарвардская** (`harv`): память команд и память данных физически разделены.

### Память команд (Instruction Memory)

| Характеристика | Значение |
|----------------|----------|
| Размер | 256 слов |
| Разрядность | 32 бита |
| Адресация | прямая, абсолютная |
| Содержимое | исполняемый код |

### Память данных (Data Memory)

| Характеристика | Значение |
|----------------|----------|
| Размер | 256 слов |
| Разрядность | 32 бита |
| Тип | однопортовая |

#### Карта памяти данных

| Диапазон адресов | Назначение |
|------------------|------------|
| 0x0000 - 0x00EF | Глобальные переменные и данные |
| 0x00F0 - 0x00FD | Стек (растёт вниз, в сторону младших адресов) |
| 0x00FE | Порт вывода (OUT, MMIO) |
| 0x00FF | Порт ввода (IN, MMIO) |

### Работа со строками (pstr)

Строки хранятся в формате **Pascal-string** (`pstr`):

```text
[длина][char0][char1][char2]...[charN]
```
## Система команд (ISA)

### Кодирование инструкций

Все инструкции имеют фиксированную длину — **1 слово (32 бита)**.

#### Формат R-тип (операции с регистрами)

``` text
 31    24 23   20 19   16 15   12 11     0
+--------+-------+-------+-------+--------+
| OPCODE |  Rd   |  Rs1  |  Rs2  |  0x000 |
+--------+-------+-------+-------+--------+
```
Где
`OPCODE` (8 бит): код операции
`Rd` (4 бита): регистр-приемник
`Rs1, Rs2` (4 бита каждый): регистры источники
`reserved` (12 бит): зарезервированное место (заполнено 0)

#### Формат I-тип (непосредственные значения)
``` text
 31    24 23   20 19   16 15                0
+--------+-------+-------+------------------+
| OPCODE |  Rd   |  Rs1  |       imm        |
+--------+-------+-------+------------------+
```
Где
`OPCODE` (8 бит): код операции
`Rd` (4 бита): регистр-приемник
`Rs1` (4 бита): регистр источник
`imm` (16 бит): непосредственное значение

#### Формат J-тип (переходы)
``` text
 31    24 23                                    0
+--------+--------------------------------------+
| OPCODE |                 imm                  |
+--------+--------------------------------------+
```
Где
`OPCODE` (8 бит): код операции (например: `JMP`, `JZ`, `JNZ`, `JL`, `JGE`, `CALL`, ...)
`imm` (24 бит): адресс перехода

#### Формат O-тип (без операндов)
``` text
 31    24 23                                    0
+--------+--------------------------------------+
| OPCODE |                0x000000              |
+--------+--------------------------------------+
```
Где
`OPCODE` (8 бит): код операции (например: `NOP`, `HALT`, `PUSH`, `POP`, `RET`)
`reserved` (24 бит): зарезервированное место (заполнено 0)

## Микрокод (Control Unit)

Управление — **микропрограммное** (`mc`). Каждая машинная инструкция реализована через последовательность микроинструкций, хранящихся в Microcode ROM.

### Структура микроинструкции

Микроинструкция содержит управляющие сигналы для всех компонентов процессора:

| Поле | Тип | Описание | Возможные значения |
|------|-----|----------|-------------------|
| `src_a` | str | Источник для входа A АЛУ | `rs1`, `rd`, `imm`, `SP`, `IN`, `OUT` |
| `src_b` | str | Источник для входа B АЛУ | `rs2`, `imm`, `SP`, `PC` |
| `alu_op` | str | Операция АЛУ | `ADD`, `SUB`, `MUL`, `DIV`, `AND`, `OR`, `XOR`, `NEG` |
| `reg_write` | bool | Запись результата в регистровый файл | `0`/`1` |
| `mem_read` | bool | Чтение из памяти данных | `0`/`1` |
| `mem_write` | bool | Запись в память данных | `0`/`1` |
| `mem_addr` | str | Источник адреса для памяти | `SP`, `IN`, `OUT`, `rs1` |
| `mem_data` | str | Источник данных для записи в память | `rs1`, `PC` |
| `use_imm` | bool | Использовать непосредственное значение | `0`/`1` |
| `write_flags` | bool | Обновить флаги ALU | `0`/`1` |
| `pc_write` | bool | Запись в PC (условный переход) | `0`/`1` |
| `cond` | str | Условие для условного перехода | `Z`, `NZ`, `G`, `GE`, `L`, `LE`, `C` |
| `jump` | bool | Безусловный переход в микропрограмме | `0`/`1` |
| `next_addr` | int/str | Следующий адрес микрокоманды | число или `IMM_JUMP`, `RET_PC`, `PSTR_HANDLER` |
| `sp_dec` | bool | Декремент SP (SP = SP - 1) | `0`/`1` |
| `sp_inc` | bool | Инкремент SP (SP = SP + 1) | `0`/`1` |

### Организация Microcode ROM

Микрокод хранится в памяти объёмом 512 записей. Адресация осуществляется следующим образом:

```text
Адрес микрокоманды = OPCODE * 4 + смещение
```
Где, `OPCODE` (8 бит): код операции, `смещение`: номер микроинструкции (например 0, 1, 2, 3)


## Ввод-вывод

Ввод-вывод реализован как **поток токенов** (`stream`).

### Модель потока токенов

В данной модели ввод и вывод представлены в виде последовательности токенов (символов), которые обрабатываются процессором последовательно. Это упрощает тестирование и отладку, позволяя заранее подготовить входные данные и проверить выходные.

**Принцип работы:**

1. При старте модели формируется буфер ввода (input buffer) из данных, указанных в конфигурационном файле
2. Процессор читает токены из буфера по запросу (инструкция IN)
3. Токены выводятся в буфер вывода (output buffer) по команде OUT
4. По окончании работы модели буфер вывода сохраняется и отображается

### Буфер ввода (Input Buffer)

**Создание буфера:**
- Буфер создаётся при инициализации модели процессора
- Данные загружаются из поля `input` конфигурационного файла
- Поддерживаются следующие форматы:
  - Строка: `"Hello, World!"` — каждый символ становится отдельным токеном
  - Список чисел: `[72, 101, 108, 108, 111]` — каждый элемент становится токеном

**Чтение из буфера:**
- Инструкция `IN Rd` читает следующий токен из буфера
- Токен извлекается из начала буфера (FIFO — первым пришёл, первым ушёл)
- После чтения токен удаляется из буфера
- При пустом буфере возвращается 0 (сигнал EOF — End Of File)
- Модель продолжает работу, но чтение возвращает 0

### Буфер вывода (Output Buffer)

**Запись в буфер:**
- Инструкция `STORE Rs1, OUT` записывает значение из регистра в буфер вывода
- Значение добавляется в конец буфера
- Буфер может хранить любое количество токенов

**Чтение из буфера:**
- По окончании моделирования все токены из буфера выводятся
- Поддерживается вывод в двух форматах:
  - Числовой: отображение числовых значений токенов
  - Строковый: преобразование чисел в символы по ASCII
